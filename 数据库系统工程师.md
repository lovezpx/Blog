### <li><font color=Yellow>协议相关知识>></font>

<font color=Pink>

> 1、PING(Packet Internet Groper)，因特网包探索器，用于测试网络连通性的程序。Ping发送一个ICMP(Internet Control Messages Protocol，因特网信报控制协议)；回声请求消息给目的地并报告是否收到所希望的ICMPecho(ICMP回声应答)。

> 3、http1.0与http1.1的区别:
>> ☆ HTTP 1.0中浏览器与服务器只保持短暂的连接，浏览器的每次请求都与服务器建立一个TCP连接，服务器完成请求处理后立即断开TCP连接，服务器不跟踪每个客户也不记录过去的请求。
>> 
>> ☆ http1.1提供永久性连接（即1.0使用非持久连接，HTTP1.0没有host的字段)。
>> 
>> ☆ HTTP 1.1中增加Host请求头字段后，实现了在一台WEB服务器上可以在同一个IP地址和端口号上使用不同的主机名来创建多个虚拟WEB站点。
>> 
>> ☆ http1.1提供身份认证（HTTP1.1提供一个基于口令的基本认证方式）。

> 4、TCP三次握手:
>> 所谓三次握手，是指建立一个TCP连接时，需要客户端和服务器总共发送3个包。
>
>> 三次握手的目的是连接服务器制定端口，建立TCP连接，并同步连接双方的序列号和确认号并交换TCP窗口大小信息。在socket编程中，客户端执行connect()时。将触发三次握手。
>
>> <li>第一次握手：建立连接时，客户端A发送SYN包（SYN=j）到服务器B，并进入SYN_SEND状态，等待服务器B确认。
>>
>> <li>第二次握手：服务器B收到SYN包，必须确认客户A的SYN（ACK=j+1），同时自己也发送一个SYN包（SYN=k），即SYN+ACK包，此时服务器B进入SYN_RECV状态。
>>
>> <li>第三次握手：客户端A收到服务器B的SYN+ACK包，向服务器B发送确认包ACK（ACK=k+1），此包发送完毕，客户端A和服务器B进入ESTABLISHED状态，完成三次握手。
>
>> 完成三次握手，客户端与服务器开始传送数据。

> 4、TCP四次挥手:
>> TCP的链接的拆除需要发送四个包，因此称为四次挥手。客户端或服务器均可主动发起挥手动作，在Socket编程中，任何一方执行close()操作即可产生挥手操作。Tcp链接是全双工的，因此每个方向都必须单独进行关闭。这原则是当一方完成它的数据发送任务后就能发送一个FIN来终止这个方向的连接。收到一个FIN只意味着这一方向上没有数据流动，一个TCP连接在收到一个FIN后仍能发送数据。首先进行关闭的一方将执行主动关闭，而另一方执行被动关闭。
>
>> <li>TCP客户端发送FIN，用于关闭客户到服务器的数据传送。
>>
>> <li>服务器收到这个FIN，它发回一个ACK，确认序号为收到的序号加1。和SYN一样，一个FIN将占用一个序号。
>>
>> <li>服务器关闭客户端的连接，发送一个FIN给客户端。
>>
>> <li>客户端发回ACK报文确认，并将确认序号设置为收到序号加1。

</font>

## <font color=Yellow>七、数据库技术基础</font>

### <font color=Yellow>基本概念>></font>

### <li><font color=Yellow>数据库系统（DataBase System,DBS）是一个采用了数据库技术，有组织地、动态地存储大量相关联数据，方便多用户访问的计算机系统。</font>

### <li><font color=Yellow>数据库管理系统（DataBase Managenebt System,DBMS）是数据库系统的核心软件，是由一组相互关联的数据的集合和一组用以访问这些数据的软件组成。其主要功能包括定义功能、数据操纵功能、数据库的运行管理和数据库的建立与维护。</font>

### <li><font color=Yellow>数据管理是处理数据的中心问题，是对数据进行分类、组织、编码、存储检索和维护。数据管理技术发展经历了三个阶段：人工管理、文件系统和数据库系统阶段。</font>

### <li><font color=Yellow>数据库系统是由计算机软件、硬件资源组成的系统，它实现了有组织地、动态地存储大量关联数据，方便多用户访问，它与文件系统的重要区别是数据的充分共享、交叉访问、与应用程序的高度独立性。</font>

### <li><font color=Yellow>数据库的四类故障是事务的内部故障、系统故障、介质故障及计算机病毒数据库恢复就是建立冗余数据，冗余是物理级的，通常认为逻辑级没有冗余。</font>

### <li><font color=Yellow>DBMS分为关系数据库系统、面向对象的数据库系统、对象关系数据库系统。</font>

### <li><font color=Yellow>数据库的三级模式结构>></font>

<font color=Pink>

> ☆ 概念模式&emsp;也称为模式，是数据库中全部数据的逻辑结构和特征的描述。它由若干个概念记录类型组成，只涉及到型的描述，不涉及到具体的值。概念模式反映的是数据库的结构和联系，所以是相对稳定的；而实例反映的是数据库某一时刻的状态，所以是相对变动的。概念模式的数据定于语言称为“模式DDL（Schema Data Definition Language）”

> ☆ 外模式&emsp;&emsp;也称用户模式或子模式，是用户与数据库系统的接口，是用户用到的那部分数据的描述。

> ☆ 内模式&emsp;&emsp;也成存储模式，是数据物理结构和存储方式的描述，是数据在数据库内部的表示方式。 


</font>

### <li><font color=Yellow>事务具有原子性、一致性、隔离性和持久性。</font>

### <li><font color=Yellow>事务的五种状态：活动状态、部分提交状态、失败状态、中止状态、提交状态。</font>

### <li><font color=Yellow>故障恢复：如何使系统出现故障后能够及时使数据库恢复到故障前的正确状态，就是数据库恢复技术。恢复的基本原理是“建立数据冗余”（重复存储）。建立冗余数据的方法是进行数据转储和记录日志文件。数据转储分为静态转储和动态转储、海量转储和增量转储。</font>

### <li><font color=Yellow>日志文件在事务处理的过程中，DBMS把事务开始、事务结束以及对数据库的插入、删除和修改的每一次操作写入日志文件。一旦发生故障，DBMS的回复子系统利用日志文件撤销事务对数据库的改变，会退到事务的初始状态。</font>

### <li><font color=Yellow>故障恢复的两个操作>></font>

<font color=Pink>

> ☆ 撤销事务(UNDO)&emsp;将未完成的事务撤销，使数据库恢复到事务执行前的正确状态。

> ☆ 重做事务(REDO)&emsp;将已经提交的事务重新执行。


</font>

### <li><font color=Yellow>并发控制是指在多用户共享的系统中，许多用户可能同时对同一数据进行操作。并发操作带来的问题是数据的不一致行，主要有三类：丢失更新、不可重复读和读脏数据。</font>

### <li><font color=Yellow>并发控制>></font>

<font color=Pink>

> ☆ 事务调度&emsp;
>> 调度：事务的执行次序。
>> 
>> 串行调度：多个事务一次串行执行。
>>
>> 并行调度：利用分时的方法同时处理多个事务。

> ☆ 并发控制的主要技术是封锁。基本封锁的类型有排它锁（简称X锁或写锁）和共享锁（简称S锁或读锁）。

</font>

### <li><font color=Yellow>数据仓库和数据挖掘基础知识</font>

<font color=Pink>

> ☆ 数据仓库&emsp;
>> 数据仓库：面向主题、数据是集成的、相对稳定的、数据反映历史变化。
>>
>> 数据仓库通常采用三层体系结构，底层为数据仓库服务器、中间层为OLAP服务器、顶层为前端工具。

</font>
